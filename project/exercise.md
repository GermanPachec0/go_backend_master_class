# Ordering and Filtering

Your [read model](https://academy.threedots.tech/knowledge/read-model) returns menu items in a fixed order, which was a good start to build an MVP.
We now need to add support for flexible sorting and filtering.

## Optional Filtering with sqlc.narg()

The pattern for optional filtering looks like this:

```sql
WHERE (sqlc.narg(restaurant_name_filter)::text IS NULL
    OR name = sqlc.narg(restaurant_name_filter)::text)
```

`sqlc.narg()` works like `sqlc.arg()` but generates a *nullable* parameter.
In Go, this produces a `*string` instead of `string`.

When you pass `nil`, the SQL parameter is `NULL`.
The `::text` cast tells PostgreSQL (and sqlc) the parameter type.
See the [sqlc named parameters docs](https://docs.sqlc.dev/en/latest/howto/named_parameters.html) for more details.

When the filter is `NULL` (not provided), the `IS NULL` check passes and the rest of the filter (after `OR`) is skipped.
When provided, it matches the restaurant name.

Note the parentheses around the whole condition.
If we added more conditions, we need them to be grouped correctly with `AND` / `OR`.

In practice, exact name match is not very user-friendly.
We can use `LIKE` with wildcards for a case-insensitive partial match:

```sql
LOWER(name) LIKE LOWER('%' || sqlc.narg(restaurant_name_filter)::text || '%')
```

## Dynamic Ordering with CASE WHEN

In the code generated by sqlc, queries have fixed `ORDER BY` clauses.
You can't concatenate strings or conditionally append clauses at runtime the way you would with a query builder.
We need another approach to support dynamic ordering.

One trick you can use is `CASE WHEN` expressions and nullable parameters.
The query stays static, but the order changes based on the values you pass in.

Here's an example we could use:

```sql
FROM orders.restaurant_menu_items mi
JOIN orders.restaurants r ON mi.restaurant_uuid = r.restaurant_uuid
ORDER BY
    CASE WHEN sqlc.narg(order_by)::text = 'price_asc' THEN mi.gross_price END ASC,
    CASE WHEN sqlc.narg(order_by)::text = 'price_desc' THEN mi.gross_price END DESC,
    CASE WHEN sqlc.narg(order_by)::text = 'name_asc' THEN mi.name END ASC,
    CASE WHEN sqlc.narg(order_by)::text = 'name_desc' THEN mi.name END DESC;
```

Each `CASE WHEN` checks the `order_by` parameter.
If the condition matches, it returns the column value for sorting.
If it doesn't, it returns `NULL`, which has no effect on the row's position.

The default ordering is a bit more complicated and needs two `CASE WHEN` clauses** (one for `restaurants.name`, one for `restaurant_menu_items.ordering`)
because PostgreSQL requires a separate expression for each sort column.

```sql
    CASE WHEN (sqlc.narg(order_by)::text IS NULL OR sqlc.narg(order_by)::text = 'default')
         THEN r.name END ASC,
    CASE WHEN (sqlc.narg(order_by)::text IS NULL OR sqlc.narg(order_by)::text = 'default')
         THEN mi.ordering END ASC,
```

We support both an empty filter and an explicit `default` value for the default ordering.

## The Go Side

In Go, you call the query as usual, passing the filter and order values:

```go
rows, err := queries.ListMenuItems(ctx, dbmodels.ListMenuItemsParams{
    RestaurantNameFilter: filter.RestaurantName,
    OrderBy:              filter.OrderBy,
})
```

All filtering and ordering logic lives in the SQL query, where PostgreSQL can optimize it.

```sql
-- name: ListMenuItemsWithRestaurant :many
-- Lists menu items with optional restaurant name filter and dynamic ordering.
-- Uses CASE WHEN to support multiple ordering options in a single query.
SELECT
    mi.restaurant_menu_item_uuid AS menu_item_uuid,
    mi.name AS menu_item_name,
    mi.gross_price,
    r.currency,
    r.restaurant_uuid,
    r.name AS restaurant_name
FROM orders.restaurant_menu_items mi
JOIN orders.restaurants r ON mi.restaurant_uuid = r.restaurant_uuid
WHERE mi.is_archived = false
  AND (sqlc.narg(restaurant_name_filter)::text IS NULL
       OR LOWER(r.name) LIKE LOWER('%' || sqlc.narg(restaurant_name_filter)::text || '%'))
ORDER BY
    CASE WHEN (sqlc.narg(order_by)::text IS NULL OR sqlc.narg(order_by)::text = 'default')
         THEN r.name END ASC,
    CASE WHEN (sqlc.narg(order_by)::text IS NULL OR sqlc.narg(order_by)::text = 'default')
         THEN mi.ordering END ASC,
    CASE WHEN sqlc.narg(order_by)::text = 'price_asc' THEN mi.gross_price END ASC,
    CASE WHEN sqlc.narg(order_by)::text = 'price_desc' THEN mi.gross_price END DESC,
    CASE WHEN sqlc.narg(order_by)::text = 'name_asc' THEN mi.name END ASC,
    CASE WHEN sqlc.narg(order_by)::text = 'name_desc' THEN mi.name END DESC;
```
